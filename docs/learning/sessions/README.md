# Session Notes

New session notes should follow the handbook-driven chapter plan in
`docs/learning/roadmap.md`.

Every chapter note should use the reusable pattern:

```text
handbook knowledge point -> RetailCare proof example -> concrete scenario
-> design reason -> tradeoff/alternative -> interview expression
```

For each important knowledge point, include a card:

```text
知识点:
中英对照:
资料依据:
资料原意:
RetailCare 例子:
具体场景:
项目证据:
为什么这样设计:
替代方案:
为什么暂时不选替代方案:
局限与后续扩展:
面试表达:
```

Suggested filename pattern:

```text
chapter-01-agent-basics.md
chapter-02-core-frameworks.md
chapter-03-rag.md
chapter-04-tool-calling.md
chapter-05-memory.md
chapter-06-multi-agent.md

```
